因为需要准确的组合，在无序数列中寻找准确的所有情况，显然是没有任何规律性的，一般算法题的思路应当是先对序列进行规律化的操作，然后再根据规律设定规则进行查找，这题利用的就是左边的数加右边的数，如果是有序数列的话，可以只进行双指针移动，因为second的数字从小到大，然后third从大到小，越小的和越大的组合一定会提前触发，不可能出现越小配合次大，越大配合次小的情况，这也就是双指针能够缩短时间的原因，因为指针移动时能够在逻辑上排除很多的情况，就像接雨水问题里面，你移动大的项可能会错过更大的机会，但只要你移动更小的项，就不会，因为这是从极端向普通拓展的过程。

## 题目

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：**答案中不可以包含重复的三元组。

**示例 1：**

```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。

```

**示例 2：**

```
输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。

```

**示例 3：**

```
输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0 。
```

## 代码

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        int n=nums.size();
        sort(nums.begin(),nums.end());
        vector<vector<int>> ans;
        for(int i=0;i<n-2;i++){
            if(i>0&&nums[i]==nums[i-1]){
                continue;
            }
            int k=n-1;
            int target=-nums[i];
            for(int j=i+1;j<n;j++){
                if(j>i+1&&nums[j]==nums[j-1]){
                    continue;
                }
                while(j<k&&nums[j]+nums[k]>target){
                    --k;
                }
                if(j==k){
                    break;
                }
                if(nums[j]+nums[k]==target){
                    ans.push_back({nums[i],nums[j],nums[k]});
                }
            }
            
        }
        return ans;
    }
};
```
